## Automated Motion Designer Loop

```text
1. BRIEF PACK - product URL, goal, audience/channel, duration, tone, references, brand cues
2. PRODUCT TRUTH - 3-10 screenshots first; recording optional for advanced demos
3. AI DIRECTION - read page/brand, infer offer, choose narrative structure and motion style
4. BEAT SHEET - decide what happens when, what the eye should notice, and where CTA lands
5. ANIMATIC - build timed scenes with draft copy, VO script, music mood, and rough layout
6. POLISH - apply device frames, transitions, easing, callouts, voice, BGM ducking, and brand
7. REVIEW - user tweaks copy, order, timing, tone, and CTA through editor/chat
8. DELIVERY - MP4/WebM/GIF/Lottie-ready variants where scoped: 16:9, 9:16, 1:1
```

Arco should feel like the user is interacting with a structured creative process, not filling out a render form.

## Quality Gate: No AI Slop

Every generated video must pass these checks before the user is asked to export:

| Check | Standard |
|-------|----------|
| Product truth | Uses only user-provided screenshots, recordings, URL, and brief. No fake UI, made-up metrics, invented customers, or unsupported claims. |
| One beat per scene | Each scene has one job: hook, problem, proof, feature, benefit, or CTA. No competing focal points. |
| Specific copy | Headlines are short, concrete, and product-relevant. Avoid vague hype such as "revolutionary", "game-changing", "unlock", or "supercharge". |
| Motion restraint | Motion helps the viewer notice the right thing. It should not decorate every object or fight the message. |
| Premium pacing | Timing, easing, overlap, transitions, VO, and music should feel intentional, not randomly assembled. |
| Reviewable process | The user can see and edit the brief, beat sheet, scenes, copy, timing, voice, music, and delivery format before export. |

If Arco cannot infer something safely, it should ask, fallback conservatively, or leave the claim out.

## Motion Presets

AI suggests **where and why**. Presets define **how it moves**.

| Preset | Purpose |
|--------|---------|
| `smooth-zoom` | Focus on the UI region that matters in the beat |
| `click-ripple` | Emphasize a click/tap moment |
| `spotlight` | Dim surround and highlight the main action |
| `title-card` | Section headline between beats |
| `feature-callout` | Text + pointer on UI |
| `transition` | Cut, fade, pan, push, or morph between scenes |

## Editor

Canva-depth, not After Effects-depth. The user should adjust story, copy, order, timing, emphasis, brand, voice, music, and export format without touching a layer timeline.

Later: keyframes and layers only if users demand them.
